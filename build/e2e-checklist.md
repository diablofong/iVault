# iVault v1.0.0 手動測試清單

> **平台：Windows only**（macOS 已於 v1.0 移除）  
> **測試前置：** `wails build` 產出 `build/bin/iVault.exe`，用正式 build 測（不是 `wails dev`）

---

## 0. 測試環境

| 項目 | 規格 |
|---|---|
| 測試機 | Windows 11 22H2 或 23H2（乾淨 VM 最佳） |
| iPhone | iOS 14 / 16 / 17 各一台（至少一台有 HEIC + MOV） |
| USB 線 | Lightning 或 USB-C |
| Apple Devices | 先測「未安裝」情境，再安裝 |

---

## 1. 首次啟動 — IDLE 變體 A

> config 不存在 / history 空 → 應顯示「初次」變體

- [ ] 雙擊 `iVault.exe`，SmartScreen 警告可點「更多資訊 → 仍要執行」
- [ ] 視窗正常出現，標題列顯示 `iVault 1.0.0`，Mica 材質背景
- [ ] IDLE 顯示：大 Logo + 主標（把 iPhone 的照片與影片備份到這台電腦）+ CTA 文字 + 三個腳註（完全離線 · 開源免費 · 原始畫質）
- [ ] **不出現**「歡迎回來」或「上次備份」字樣
- [ ] 深色模式：跟隨 Windows 系統主題（設定→色彩→深色）
- [ ] 淺色模式：切換系統主題後即時更新（不需重啟 app）
- [ ] 語言切換 EN↔中，所有文字正確翻譯
- [ ] 語言選擇持久化（關閉重開仍維持）

## 2. Apple Devices 未安裝

> 在 **未安裝** Apple Devices 的機器上測

- [ ] 啟動後出現 DRIVER_MISSING 頁，**不是** IDLE 頁
- [ ] 顯示 Apple 圖示 + 「需要先安裝 Apple Devices（免費）」文案
- [ ] 顯示「一鍵安裝」按鈕
- [ ] 底部「為什麼需要？」可展開/收合
- [ ] 「一鍵安裝」→ 啟動 winget（有 winget 時）或開 Microsoft Store
- [ ] 安裝完成後顯示「安裝完成！請重新插拔 iPhone」
- [ ] **沒有** `driver:required` / `AMDS` 等技術詞彙出現在 UI

---

## 3. Apple Devices 已裝未啟動

> Apple Devices 安裝但 AMDS service 未跑

- [ ] 顯示的引導文案與「未安裝」**不同**（已裝未啟動的差異化文案）
- [ ] 20 秒後顯示 `AMDS_TIMEOUT` 錯誤（人話：「無法啟動 Apple 裝置服務，請重新插拔 iPhone 再試」）

---

## 4. 首次啟動引導（三步驟）

> history = 0，插上 iPhone 後的 onboarding

- [ ] 步驟一：確認 Apple Devices 驅動已就緒
- [ ] 步驟二：選擇預設備份路徑（Folder Picker 正常）
- [ ] 步驟三：詢問是否開機自動啟動
  - 點「是」→ Windows 登錄寫入，工作管理員「啟動」頁可看到 iVault
  - 點「否」→ 正常進入 READY，不寫登錄

## 5. 裝置連線 — TRUST_GUIDE

- [ ] 插入**未配對** iPhone → 顯示 TRUST_GUIDE
- [ ] 畫面顯示等待文案 + spinner
- [ ] 10 秒無進度 → 淡入提示「iPhone 螢幕是否亮著？信任視窗有時需要解鎖手機才會出現」
- [ ] 在 iPhone 點「信任」→ 自動推進 READY（有確認動畫）
- [ ] 60 秒超時（不點信任）→ 顯示「似乎沒收到信任回應，請拔掉重插 iPhone 試試」+ 重試按鈕
- [ ] 重試按鈕 → 回 IDLE

---

## 6. 裝置連線 — READY

- [ ] 插入**已配對** iPhone → 直接進入 READY
- [ ] READY 顯示：裝置名稱、照片數量、可用磁碟空間、預估備份大小
- [ ] 首次：顯示「尚未備份過」
- [ ] 有增量：顯示新照片/影片數
- [ ] 磁碟空間：估算含最大單檔（不只平均）
- [ ] iCloud 最佳化儲存（裝置上有縮圖版本）→ 顯示 iCloud 提示
- [ ] 充電線（USB-A 舊款，非資料線）→ 顯示差異化提示「請改用資料傳輸線」

---

## 7. 自動備份倒數（回訪）

> 先完成一次備份，再重新插上 iPhone

- [ ] 插上 iPhone 後出現 3 秒倒數，三個按鈕：`[立即開始]` `[稍後 15 分]` `[這次跳過]`
- [ ] 倒數結束 → 自動開始備份
- [ ] 點「稍後 15 分」→ 倒數消失，15 分後再次詢問
- [ ] 點「這次跳過」→ 停留 READY，等手動按
- [ ] 設定關閉「自動備份」→ 回訪時不出現倒數

---

## 8. 備份進行中 — BACKING_UP

- [ ] 進度條正常推進
- [ ] 顯示「正在備份 YYYY 年 M 月的回憶 · 第 X / Y 個」（含月份故事感）
- [ ] 沒 EXIF 的檔案 fallback：「正在備份第 X / Y 個」
- [ ] 顯示速度（MB/s）+ 預估剩餘時間 ETA
- [ ] 長時間備份：出現心理安撫文案「首次備份需較長時間，之後只備份新照片」
- [ ] 備份中途**阻止系統睡眠**（把螢幕超時設 1 分鐘，確認不睡眠）
- [ ] 最小化後繼續備份（不中斷）
- [ ] 視窗關閉（X）→ 出現確認框「備份進行中，確定要結束嗎？」

---

## 9. 拔線中斷

- [ ] **備份中拔線** → 3 秒內回 IDLE，顯示「中斷變體」
- [ ] 中斷 IDLE 顯示：「你的上次備份被打斷了」+ 已備份數 / 總數 + 「進度安全保留」
- [ ] 重新插上 iPhone → READY 顯示「繼續備份」，從斷點繼續
- [ ] 已備份的檔案**不重複下載**

---

## 10. 手機沒電斷線

- [ ] 手機電量耗盡自動斷開 → 顯示專屬文案「手機好像沒電了，充電後再試」（與一般拔線不同）

## 11. 備份完成 — DONE 頁

### 首次備份
- [ ] 顯示「這是你用 iVault 備份的第一次。歡迎。」（首次彩蛋）
- [ ] 主標：「你的 X 張照片和 Y 段影片安全了」
- [ ] Bento 統計四格：照片 / 影片 / 空間 / 花了（用溫暖量詞）
- [ ] 副標：「備份於 YYYY/MM/DD · 共 X.X GB」
- [ ] 主按鈕（大）：「開啟備份資料夾 →」可正常開 Explorer
- [ ] 次按鈕：「完成」回 IDLE

### 回訪備份
- [ ] **不出現**「第一次備份」彩蛋
- [ ] 主標：「新增 X 張照片和 Y 段影片」
- [ ] 贊助文：「iVault 永遠免費開源。如果它幫到你，請我喝杯咖啡 →」（極小字、無 emoji）

## 13. HEIC + 影片日期分類

> 使用有 HEIC 照片（iPhone 預設）和 MOV 影片的 iPhone

- [ ] 備份後檔案按 `YYYY-MM/` 分類，HEIC 照片放到**拍攝月份**（不是備份當月）
- [ ] MOV / MP4 影片放到**拍攝月份**（QuickTime mvhd 解析）
- [ ] DONE 頁：照片和影片**分開統計**顯示
- [ ] 無法解析 EXIF 的檔案進入 `_unknown-date/` 資料夾
- [ ] DONE 頁：`unknownDateCount > 0` 時顯示說明文案

## 14a. 多設備情境（家庭用，同一台電腦）

> 需要兩台 iPhone（Device A、Device B）

**情境 1：Device A 備份完成 → Device B 第一次插入**
- [ ] IDLE 顯示 Device A 的 returning 資訊（含 Device A 名稱）
- [ ] Device B 插入後，READY **不出現** 3 秒自動備份倒數（B 是第一次）
- [ ] DONE 頁顯示 Device B 的「首次彩蛋」

**情境 2：Device A 中斷（119/141）→ Device B 插入**
- [ ] IDLE **不顯示**「你的上次備份被打斷了 119/141」
- [ ] READY 的 last-backup-row **不顯示**中斷狀態
- [ ] 3 秒倒數行為依 Device B 自身歷史決定（首次→不倒數，回訪→倒數）

**情境 3：Device A 中斷 → Device A 重新插入**
- [ ] IDLE 顯示「你的上次備份被打斷了 119/141」
- [ ] READY 顯示中斷狀態
- [ ] 從斷點繼續備份，不重複下載

**情境 4：Device A 中斷 + Apple Devices 被砍（截圖場景）**
- [ ] IDLE 顯示中斷變體（保留進度數字）
- [ ] CTA 文字變為「先安裝 Apple Devices，才能繼續備份」（**不是**「重新連接 iPhone」）
- [ ] 安裝好 Apple Devices 後，CTA 文字恢復正常
